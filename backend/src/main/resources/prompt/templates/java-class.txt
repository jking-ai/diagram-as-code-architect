MERMAID SYNTAX RULES (you MUST follow these exactly):
1. Start with "classDiagram" on the first line.
2. Class definition: class ClassName
3. Add members with: ClassName : +publicMethod() ReturnType or ClassName : -privateField Type
4. Visibility prefixes: + public, - private, # protected, ~ package-private.
5. Relationships:
   - Inheritance: ChildClass --|> ParentClass
   - Implementation: Class ..|> Interface
   - Association: ClassA --> ClassB
   - Dependency: ClassA ..> ClassB
   - Aggregation: ClassA o-- ClassB
   - Composition: ClassA *-- ClassB
6. Annotations: <<interface>>, <<abstract>>, <<enumeration>> on their own line after class declaration.
7. NEVER use special characters in class names — only letters, numbers, underscores.
8. NEVER put spaces in type names. Use generic notation carefully: List~String~ not List<String>.
9. Cardinality labels: ClassA "1" --> "*" ClassB : contains
10. NEVER use the word "end" or "class" as a class name without a suffix.
11. Keep diagrams focused — show important classes and relationships, omit trivial getters/setters.
12. Each statement must be on its own line.
