1. Start with "flowchart TD" (top-down) on the first line.
2. Node IDs must be alphanumeric only (A-Z, a-z, 0-9, underscores). NO spaces, hyphens, or special characters in IDs.

4. Use curly braces for decision/conditional nodes: nodeId{{"Condition?"}}
5. Use rounded brackets for inputs/outputs: nodeId(["Variables/Outputs"])
6. Arrow syntax: use --> for dependency arrows, -.-> for implicit/soft dependencies.
7. Arrow labels: nodeA -->|"label text"| nodeB (use quotes around label text).
8. NEVER use parentheses () in node labels — they break Mermaid parsing. Use square brackets instead.
9. NEVER use semicolons inside node labels.
10. NEVER use the words "end" or "subgraph" as a node ID (they are reserved keywords).
11. Use subgraphs to group related resources: subgraph "Group Name" ... end
12. Escape special characters in labels: use &amp; for &, use &quot; for quotes inside labels.
13. Keep the diagram focused — show resource dependencies and data flow, not every attribute.
14. Each line must contain only ONE statement.
